Rules

This simulation models a 'gaming run' using a simplified coin-flipping game between two players, A and B.

Gaming Run Setup:

  • Each player begins with an equal number of 'coins,' their initial wallet limit.
  • Players set their 'bet' for each game before the run begins.
  • A gaming run consists of a predetermined maximum number of games.
  • The run ends early if a player's coin count falls below their bet.

Game Mechanics:

  • Each game consists of multiple 'coin flips.'
  • 'Heads': Player A's coin count decreases by one, and Player B's increases by one.
  • 'Tails': Player B's coin count decreases by one, and Player A's increases by one.
  • A player loses when their coin count reaches zero.
  • A game ends in a draw if a maximum number of coin flips is reached.
  • In case of a win or loss, the bet amount is transferred from the loser to the winner. In case of a draw, no coins are transferred.
